# 仕様書

## 1. 目的

Web小説を快適に閲覧するためのiPhone向け読書アプリを提供する。

## 2. 対象プラットフォーム

- iOS
- iPhone優先
- 将来的なiPad対応は未定

## 3. 対応サイト

初期リリース:

- 小説家になろう系

将来候補:

- カクヨム
- ハーメルン
- その他Web小説サイト

## 4. 登録フロー

1. Safariで作品ページを開く
2. 共有メニューを開く
3. 本アプリを選択
4. 作品を登録
5. メタ情報取得
6. 本棚へ追加

## 5. クロール仕様

- フォアグラウンドのみ実行
- URLSession使用
- ログイン不要
- robots.txtおよび利用規約は別途確認
- 作品情報取得
- 話一覧取得
- 本文取得

## 6. Parser

実装方式:

- Swift
- SwiftSoup

特徴:

- Parserはアプリ内蔵
- 外部配信なし
- アプリ更新で対応

## 7. データ保存

DB:

- SQLite
- GRDB

保存対象:

- 作品情報
- エピソード情報
- 本文
- 読書位置
- 更新日時

## 8. Reader仕様

### 表示

- 縦書き必須
- Kindle風UI
- 横フリックページ送り
- タップページ送り

### レンダリング

- WKWebView
- CSS writing-mode: vertical-rl
- CSS columns
- scroll-snap

### 読書位置

保存内容:

- 作品
- 話
- ページ位置

### テーマ

- システムに合わせる
- ライト
- ダーク
- セピア（将来）

### ユーザー設定

- 上下左右マージン
- フォント
- 文字サイズ
- 行間幅

## 9. 書庫画面

表示項目:

- タイトル
- 作者
- 保存話数
- 最終更新日時
- 使用容量

集計:

- 総ストレージ使用量
- 作品別使用量

## 10. オフライン動作

可能:

- 本棚閲覧
- 本文閲覧
- 検索（将来）

不可:

- 更新チェック
- 新規取得

## 11. 非機能要件

### パフォーマンス

- 起動3秒以内
- ページ送り100ms以内

### 保守性

- Site Adapter分離
- Parser分離
- Reader分離

### セキュリティ

- ローカル保存のみ
- 外部サーバ依存なし

## 12. MVP

初期リリース対象:

- Safari共有登録
- なろう対応
- ローカル保存
- 縦書きReader
- ページ送り
- 読書位置保存
- ストレージ表示
